# DeepSeek 评估：CC长久记忆架构

## 发现的缺陷

### 1. 图谱检索依赖脆弱
[[wikilinks]] 在知识库初期文档稀疏时链接密度低，图谱几乎空跑。
建议: 扩充为共现词图谱（同文档高频共现词自动建立隐式边）。

### 2. 三路融合权重未定义
RRF k=60 统一对待三路，但不同标签类型的记忆适合不同路。
建议: #mem-rule 偏BM25，#mem-issue 偏向量，分标签类型调参。

### 3. 缺少数值型记忆检索
"超时=3000ms"这类精确数值，向量和BM25都容易漏。
建议: 从 #mem-rule 自动抽取参数名=值对，建立轻量键值索引。

### 4. 水位估算精度未验证
字符数估算token对中英文差异巨大（中文1.5-2，英文0.25），混合场景误差>30%。
建议: 区分中英文分别估算，或让Claude传原文片段给脚本用tiktoken精算。

### 5. 缺少写入前主动冲突检测
目前只有读取时被动冲突处理，蒸馏时如果与已有决策矛盾不提醒。
建议: 写入前用Reranker检查新旧结论的语义矛盾（相似度低但主题同=冲突信号）。

### 6. 健康监控未定义告警阈值
health.py 只管跑，不管"什么算不健康"。
建议: 定义3-5个关键指标及告警线。

## 改进建议

### 1. 增加 /forget 遗忘机制
证伪记忆应可主动标记 #mem-obsolete 永不召回，而非仅靠时间老化。

### 2. 启动记忆加载改为主题感知
不要启动即加载所有 #hot，等用户第一条消息后再用水位判断决定加载方式。

### 3. /digest --inject 注入位置需精细控制
注入的摘要应放在A（实时对话）和B（已有快照）之间，加明确分隔标记。

### 4. 冷启动引导机制
首次使用时主动提示明确技术栈、核心模块、当前待办，快速建立初始记忆骨架。

### 5. 记忆遗忘曲线连续衰减
用 e^(-λt) 替代 hot/warm/cold 三档，半衰期30天，让衰减更平滑。
